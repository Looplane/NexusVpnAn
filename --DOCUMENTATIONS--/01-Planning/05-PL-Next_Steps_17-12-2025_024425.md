### Priority 1: Deploy to Production ‚ö°

You have two options:

#### Option A: Automated Deployment (Recommended)

This will guide you through the entire process step-by-step.

#### Option B: Manual Deployment
Follow the quick guide: **[05-DEP-Quick_Cloud_Deployment_17-12-2025_024425.md](../06-Deployment/05-DEP-Quick_Cloud_Deployment_17-12-2025_024425.md)**

### Deployment Checklist
1. **Setup Supabase Database** (5 minutes)
   - [ ] Create Supabase project
   - [ ] Run migration: `setup_db.sql` or use `./infrastructure/setup-supabase-db.sh`
   - [ ] Get connection string

2. **Deploy Backend to Render** (10 minutes)
   - [ ] Connect GitHub repository
   - [ ] Render auto-detects `render.yaml`
   - [ ] Link database or set `DATABASE_URL`
   - [ ] Deploy and note backend URL

3. **Deploy Frontend to Vercel** (5 minutes)
   - [ ] Import from GitHub
   - [ ] Set root: `frontend`
   - [ ] Set `VITE_API_URL`: `https://your-backend.onrender.com/api`
   - [ ] Deploy and note frontend URL

4. **Update CORS** (2 minutes)
   - [ ] Update `FRONTEND_URL` in Render
   - [ ] Update `CORS_ORIGIN` in Render
   - [ ] Backend auto-redeploys

5. **Verify Deployment** (5 minutes)
   - [ ] Run: `./infrastructure/test-cloud-deployment.sh`
   - [ ] Test login flow
   - [ ] Check admin panel

**Total Time**: ~25-30 minutes
